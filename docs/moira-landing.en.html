// what Moira is

Moira isn't another agent. It's a cockpit that drives the best agents across the whole software lifecycle — with human quality gates and an undeniable, git-native audit trail.

We don't re-implement an agent engine. We orchestrate pluggable backends (Claude Code, Codex, local models) and add the layer they lack: governance, traceability and proof. All of it lives in your repository — because the truth about a system should live where its code does.

Philosophy

Four principles that keep it all in check

01

Govern, don't replace

Moira doesn't compete with the agent — it governs it. Frontier models and local engines do the heavy lifting; Moira makes sure every step is intentional, documented and reversible.

02

A human at the gates

Between checkpoints the agent runs on its own. At every checkpoint sits a gate: automatic, hybrid (confidence-routed) or human. You decide where the machine may pass alone, and where a person is needed.

03

Git-native, tamper-evident

Every decision, cost and approval lands in the repo as an audit record chained by hashes. History can't be quietly rewritten — tampering is detectable.

04

It all starts with intent

Before a single line of code, you shape intents, requirements and specs — by driving your AI SDLC skills. Code is just the consequence of a well-stated “why”.

Compliance

Compliance as a first-class quality eval

A corpus of regulations (GDPR, WCAG 2.2, NIS2, ISO 27001, AI Act, DSA, energy…) lives in the repo as standards. Lightly-grouped auditor agents (privacy / security / accessibility / AI Act / sector + an orchestrator) read the code read-only and return a scorecard of findings: severity, the regulation article, file:line, recommendation.

  • On-demand audit: “🛡 Compliance” against a chosen regulation
  • BLOCKER/HIGH cap the score — a clear “do not merge” signal
  • Every audit is a normal, audited run in the hash chain
